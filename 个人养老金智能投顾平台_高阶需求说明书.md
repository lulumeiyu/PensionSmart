## 1. 项目定位与目标

### 1.1 项目定位

面向 C 端个人用户的一站式养老金智能投顾平台，围绕“获客 -> 教育 -> 规划 -> 投资 -> 陪伴”构建业务闭环。

### 1.2 核心目标

- 解决“开户热、缴存冷”的转化断层
- 提供可解释、可执行、可审计的投顾服务
- 在合规前提下建立长期留存与 AUM 增长能力

### 1.3 平台约束

- 金融交易“正确性优先于功能丰富度”
- 合规门禁优先于上线速度
- MVP 坚持范围冻结，避免需求扩散

---

## 2. MVP 范围冻结（P0）

### 2.1 首期仅交付 3 条主链路

1. **税优获客链路**  
   H5 税优计算器 -> 注册引导 -> 渠道归因

2. **开户注册链路**  
   手机号注册 -> 实名/KYC -> 养老金资金账户开立

3. **投顾首投链路**  
   风险测评 -> 推荐方案(3档) -> 组合一键申购 -> 持仓查看

### 2.2 MVP In / Out

| 模块 | In（本期） | Out（后续） |
|------|------------|-------------|
| 获客 | 税优计算器、渠道追踪 | 社交裂变任务体系 |
| 用户 | 注册、实名、KYC、单银行开户 | 多银行并行开户、会员体系 |
| 投顾 | 标准问卷风测、TRF 三档展示 | AI 对话测评、TDF/BL 高级策略 |
| 交易 | 组合一键申购、幂等与回滚 | 转换交易、自动调仓 |
| 投后 | 首版持仓与收益展示 | 智能再平衡自动执行 |

### 2.3 MVP 退出标准（DoD）

- 三条主链路端到端成功率 >= 95%（样本 n >= 200）
- 适当性拦截覆盖率 = 100%
- 日对账差错率 <= 0.01%（按笔数）
- 审计日志关键事件完整率 = 100%
- 阻断级缺陷（资金/合规/主链路）= 0

## 4. 服务边界与关键接口（MVP）

### 4.1 服务边界

1. `user-service`：注册、实名、KYC、用户状态  
2. `pension-account-service`：养老金开户与状态同步  
3. `advisor-service`：风险测评、风险等级、组合建议  
4. `product-service`：产品主数据、风险等级、费率  
5. `trade-service`：申购下单、状态流转、补偿  
6. `recon-service`：清算、对账、差异工单  
7. `message-service`：交易通知、合规提醒  
8. `audit-service`：审计留痕查询

## 6. 金融正确性技术策略

### 6.1 幂等策略

- 客户端 + 服务端双层幂等  
- 幂等键唯一约束，24 小时窗口可配置  
- 幂等命中返回原订单，不重复下单

### 6.2 事务与补偿策略

- 组合交易采用 Saga 编排  
- 任一子单失败触发补偿  
- 状态机单向流转，禁止逆向覆盖

### 6.3 对账策略

- 对账维度：笔数、金额、份额、手续费、状态  
- 频率：T+1 日终对账 + 关键时段增量核对  
- 差异处理：自动建单并分级（P0 资金、P1 状态）

### 6.4 审计留痕

关键事件（实名变更、风测、适当性决策、交易操作、用户确认）必须实时留痕，确保可检索、可追溯、不可篡改。

## 7. 验收口径（可测化）

### 7.1 统一统计口径

- 统计窗口：自然日（UTC+8）
- 指标输出必须包含样本量 n
- 默认报告字段：P95/P99/成功率/差错率

### 7.2 指标修正

| 指标 | 原理想口径 | 可测口径 |
|------|------------|----------|
| 数据一致性 | 差异率 = 0 | 日对账差错率 <= 0.01%，资金差异 100% 当日建单处理 |
| 方案时延 | <= 3 秒 | P95 <= 3 秒，P99 <= 5 秒，超时降级基础方案 |
| 触达时延 | <= 3 秒 | P95 <= 3 秒，P99 <= 10 秒，超时补发 |
| 产品准确率 | 100% | 主数据 >= 99.9%，关键合规字段 = 100% |

## 10. 灰度与应急

### 10.1 灰度发布

- 白名单 -> 小流量 -> 扩容发布  
- 灰度期启用更严格资金链路告警阈值  
- P0 资金告警触发自动暂停新增交易入口

### 10.2 应急预案

1. 交易通道异常：切备用通道或熔断并公告  
2. 对账异常：冻结相关批次，启动差错流程  
3. 数据安全事件：隔离、审计、上报、通知

## 15. 风险优先级与责任矩阵（新增）

### 15.1 风险分级定义

- **P0**：资金安全、合规违规、主链路不可用
