## 一、项目背景与定位

### 1.1 宏观背景

中国正在经历全球规模最大、速度最快的人口老龄化进程。截至2025年末，60岁以上人口已突破3亿，占总人口比重超过21%。与此同时，基本养老保险替代率持续承压——全国平均水平已从2000年的约70%下降至当前的约40%，远低于国际劳工组织建议的55%最低替代率。这意味着仅靠第一支柱（基本养老保险）已无法维持退休前的生活水平，第三支柱（个人养老金）的战略重要性日益凸显。

2022年11月，个人养老金制度在36个城市先行启动；2024年扩大至全国范围。制度设计采用EET税收优惠模式（缴费免税、投资免税、领取按3%单独计税），每人每年缴费上限12,000元。截至2025年末，全国开户人数已超7,000万，但一个突出的结构性问题是——**实际缴存比例仅约30%**，大量用户处于"开了户但不知道怎么用"的状态。

这一"开户热、缴存冷"的局面，恰恰揭示了一个深层需求缺口：用户不是不想存养老金，而是缺乏"存什么、怎么存、怎么管"的专业指导。传统的银行网点、基金销售平台提供的多是货架式产品罗列，而非以用户退休目标为导向的配置建议。这构成了本项目的核心市场机会。

### 1.2 行业痛点

从用户视角来看，当前个人养老金投资面临四个层级的痛点：

**认知层**——多数用户对第三支柱的制度框架、税优机制、产品品类缺乏基本了解。"个人养老金和社保养老金有什么区别？""我缴存12,000元到底能省多少税？""基金、理财、保险、储蓄这四类产品我该怎么选？"——这些基础问题尚无一个权威、易懂、一站式的解答渠道。

**决策层**——目前市场上可投资的个人养老金产品超过700只（含养老目标基金、养老理财产品、商业养老保险、养老储蓄存款），风险收益特征各异。普通用户面对如此庞大的选择空间，极易陷入决策瘫痪，最终要么不做选择（资金沉淀在活期），要么跟风买入"排行榜第一"（未必适合自己）。

**管理层**——养老金投资是跨越20-30年的长期过程，需要根据市场变化、个人生活事件（结婚、生育、换工作）和年龄增长不断调整配置。但绝大多数用户在"买入"之后便处于"放养"状态，缺乏持续的再平衡和组合优化。

**信任层**——传统渠道（银行理财经理、保险代理人）以产品销售为导向，推荐的往往是佣金最高的产品而非最适合用户的产品。这种利益冲突导致用户对"被推荐"的方案天然不信任。

### 1.3 项目定位

本项目定位为：**面向C端个人用户的一站式养老金智能投顾平台**。

平台的核心使命是"让每个人都拥有专业、透明、可信赖的养老财务规划"。区别于现有市场参与者，平台强调三个定位锚点：

**垂直深耕养老金赛道**——不做全品类理财平台，100%聚焦个人养老金场景。从知识教育、产品筛选到资产配置、持仓管理，全链路围绕"退休"这一终极目标展开。

**AI原生、顾问导向**——以AI算法为核心驱动力（而非传统的人海销售模式），站在用户利益角度提供配置建议（而非站在产品端推销），从根本上解决信任层痛点。

**营销与投顾双轮驱动**——不只做好"投"（智能配置），也做好"顾"（精准触达、持续陪伴），形成"获客 → 教育 → 规划 → 投资 → 陪伴"的完整闭环。

### 1.4 项目目标

| 维度 | 目标 | 时间框架 |
|------|------|----------|
| 用户规模 | 注册用户500万，有效投资用户150万 | 3年 |
| 资产规模（AUM） | 管理养老金资产200亿元 | 3年 |
| 用户满意度 | NPS（净推荐值）≥ 40 | 持续 |
| 合规资质 | 取得或依托持牌机构的基金投顾/销售牌照 | 上线前 |
| 技术能力 | 系统可用性 ≥ 99.9%，投顾方案生成 ≤ 3秒 | 上线后 |

## 二、用户分析

### 2.1 目标用户群定义

平台的目标用户是"有养老储备意识和行动意愿的城镇职工"。这个群体可以进一步用三个维度切分：

**人口统计维度**：年龄25-55岁，月收入8,000元以上（个税起征点以上才有税优价值），在一二线城市或经济发达的三线城市工作。

**金融素养维度**：从"只知道余额宝"的入门级到"有基金和保险投资经验"的进阶级，覆盖广泛的认知光谱。

**养老规划阶段维度**：从"还没想过养老"的启蒙期，到"想规划但不知道怎么开始"的探索期，到"已经在投但缺乏管理"的执行期。

## 三、业务域分解

### 3.1 业务域全景

平台的业务按照领域驱动设计（DDD）的思想划分为**6个核心业务域**和**2个支撑业务域**。核心业务域直接承载商业价值，支撑业务域为核心域提供基础能力。

│                                                                     │
│    ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐            │

│    │ 营销    │→ │ 用户    │→ │ 投顾    │→ │ 产品    │            │
│    │ 获客域  │  │ 管理域  │  │ 引擎域  │  │ 中心域  │            │
│    └─────────┘  └─────────┘  └─────────┘  └─────────┘            │
│         │                                          │               │
│         │            ┌──────────┐    ┌──────────┐  │               │
│         └───────────▶│ D5       │    │ D6       │◀─┘               │
│                      │ 交易     │    │ 内容     │                   │

│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

### 3.2 各域定义与重要度分析

#### D1 · 营销获客域

| 维度 | 说明 |
|------|------|
| **定义** | 负责从外部流量中获取目标用户，并通过精准触达策略推动用户沿生命周期漏斗向下转化 |
| **核心价值** | 用户增长的发动机；直接决定平台的获客成本（CAC）和增长天花板 |
| **重要度** | ★★★★★（5/5）—— 没有用户，一切无从谈起 |
| **业务挑战** | 养老金融是低频、低感知的品类，用户不会主动搜索"养老金"；需要找到高转化率的触达场景和传播机制 |

**子域分解：**

| 子域 | 定义 | 优先级 |
|------|------|--------|
| D1.1 用户画像与标签 | 构建多维用户标签体系（基础属性、行为属性、金融属性、生命周期属性），为精准营销和投顾提供数据基础 | P0 |
| D1.2 精准触达引擎 | 基于事件/时间/算法三种触发机制，通过App推送、短信、微信等渠道实现个性化触达 | P0 |
| D1.3 营销自动化工坊 | 可视化编排多步骤营销旅程（如注册→等2小时→未开户→推送引导视频） | P1 |
| D1.4 A/B测试平台 | 对文案、落地页、推送时机等变量进行对照实验，自动分配流量和统计显著性 | P1 |
| D1.5 社交裂变工具 | "邀请好友测退休金""组队存养老金"等利用社交关系链的低成本获客机制 | P2 |
| D1.6 税优计算器（获客工具） | 用户输入收入后即时展示节税金额，支持H5独立分享，是核心获客入口 | P0 |

#### D2 · 用户管理域

| 维度 | 说明 |
|------|------|
| **定义** | 管理用户从注册到流失的全生命周期，包括账户体系、会员体系、消息中心 |
| **核心价值** | 用户资产的载体；用户数据和关系的归口管理方 |
| **重要度** | ★★★★☆（4/5）—— 基础设施级别，做不好会影响所有上层业务 |
| **业务挑战** | 养老金账户涉及银行、基金公司等多方系统对接，账户体系复杂度高 |

**子域分解：**

| 子域 | 定义 | 优先级 |
|------|------|--------|
| D2.1 账户注册与认证 | 手机号注册、实名认证（姓名+身份证+人脸）、KYC信息采集 | P0 |
| D2.2 养老金账户管理 | 对接银行开立个人养老金资金账户，支持跨行归集和账户状态管理 | P0 |
| D2.3 会员等级体系 | 青铜→白银→黄金→铂金→钻石五级，与资产规模和活跃度挂钩，高等级享费率折扣和专属权益 | P1 |
| D2.4 消息通知中心 | 统一管理所有触达消息（系统通知、交易确认、投顾建议、营销推送、合规提醒），支持精细化开关控制 | P0 |
| D2.5 用户偏好管理 | 管理用户的投资偏好（ESG、行业偏好）、通知偏好、界面偏好等个性化设置 | P1 |

#### D3 · 投顾引擎域

| 维度 | 说明 |
|------|------|
| **定义** | 平台的"大脑"，负责理解用户风险特征并生成个性化的资产配置方案，以及持续的再平衡管理 |
| **核心价值** | 平台核心竞争壁垒所在；是用户愿意付费的"专业价值"的直接体现 |
| **重要度** | ★★★★★（5/5）—— 没有投顾能力就只是一个产品货架，和传统平台无差异 |
| **业务挑战** | 配置算法需兼顾学术严谨性和用户体验，且需通过合规审查（投顾建议的适当性义务） |

**子域分解：**

| 子域 | 定义 | 优先级 |
|------|------|--------|
| D3.1 风险测评引擎 | 双维度评估：客观承受能力（年龄、收入、经验等量化）+ 主观承受意愿（情境化问卷）。输出C1-C5五个风险等级。支持标准问卷和AI对话两种测评模式 | P0 |
| D3.2 资产配置引擎 | 核心算法模块。支持目标日期策略（TDF，基于下滑曲线）、目标风险策略（TRF，恒定风险水平）、均值-方差优化+Black-Litterman修正三种方法论 | P0（TRF）/ P1（TDF+BL） |
| D3.3 组合推荐引擎 | 基于配置结果推荐具体产品组合，至少提供保守/推荐/进取三个方案，展示预期收益区间、最大回撤、夏普比率、配置饼图 | P0 |
| D3.4 动态再平衡引擎 | 定期检测（季度）+ 事件驱动（市场剧烈波动、用户生活变化、政策变化），偏离超5个百分点时推送再平衡建议 | P1 |
| D3.5 行为金融学修正 | 根据用户行为特征（频繁查收益=损失厌恶、恐慌赎回=处置效应）动态调整推荐策略，避免推荐"拿不住"的组合 | P2 |
| D3.6 回测与模拟引擎 | 基于历史数据对配置方案进行回测，展示"如果10年前这样配"的收益曲线，增强用户信心 | P1 |

#### D4 · 产品中心域

| 维度 | 说明 |
|------|------|
| **定义** | 管理平台上可供用户投资的全部养老金融产品，包括产品接入、信息展示、筛选比较 |
| **核心价值** | 投顾方案的"弹药库"；产品丰富度和信息透明度直接影响用户决策质量 |
| **重要度** | ★★★★☆（4/5）—— 产品不够丰富，投顾引擎"巧妇难为无米之炊" |
| **业务挑战** | 需与基金公司、理财子公司、保险公司等多类机构对接产品数据，标准化难度大 |

**子域分解：**

## 四、业务关联关系分析

### 4.1 域间依赖关系

八大业务域之间并非孤立存在，而是通过数据流、事件流和控制流形成有机的协作网络。以下从三个视角分析域间关系。

#### 4.1.1 数据流向

```
    ┌──────────┐           ┌──────────┐
    │ S1 数据  │◀──采集────│ 全部域   │
    │   中台   │──画像/标签─▶ D1,D3   │
    └──────────┘           └──────────┘
         │
         │ 指标/报表
         ▼
    ┌──────────┐
    │ 管理后台 │
    └──────────┘

    D1 营销获客 ──用户基础信息──▶ D2 用户管理
         │                            │
         │ 触发测评                    │ 风险等级
         ▼                            ▼
    D3 投顾引擎 ◀──风险画像────────── D2
         │
         │ 配置方案（产品+权重）
         ▼
    D4 产品中心 ──产品详情/净值──▶ D3（用于配置计算）
         │                            │
         │ 产品信息                    │ 购买指令
         ▼                            ▼
    D5 交易结算 ──持仓/收益数据──▶ D2（更新用户资产）
         │                       ──▶ D3（用于再平衡判断）
         │                       ──▶ D1（触发营销事件）
         ▼
    D6 内容教育 ──学习行为数据──▶ D1（丰富用户画像）
```

#### 4.1.2 事件驱动关系

平台采用事件驱动架构（EDA），各域通过领域事件实现松耦合的异步协作。核心事件链路如下：

**链路一：新用户首投链路**

```
[D1] UserRegisteredEvent          → 触发税优计算器推荐
       ↓
[D2] RealNameVerifiedEvent        → 触发养老金开户引导
       ↓
[D2] PensionAccountOpenedEvent    → 触发风险测评邀请
       ↓
[D3] RiskAssessmentCompletedEvent → 触发配置方案生成
       ↓
[D3] AllocationRecommendedEvent   → 触发方案展示 + 推送
       ↓
[D5] FirstPurchaseCompletedEvent  → 触发"恭喜"消息 + 定投引导
       ↓
[D1] UserActivatedEvent           → 用户进入"活跃"生命周期阶段
```

**链路二：再平衡触发链路**

```
[S1] MarketVolatilityDetectedEvent   → 市场单日跌幅 > 3%
       ↓
[D3] RebalanceEvaluationTriggered    → 评估所有受影响组合
       ↓
[D3] RebalanceSuggestionGenerated    → 为偏离 > 5% 的组合生成建议
       ↓
[D2] NotificationSentEvent           → 推送"您的组合需要再平衡"
       ↓
[D5] RebalanceExecutedEvent          → 用户确认后执行调仓
       ↓
[D3] PortfolioUpdatedEvent           → 更新组合状态
```

**链路三：营销转化链路**

```
[D1] TaxCalculatorUsedEvent          → 用户使用税优计算器但未注册
       ↓
[D1] RegistrationReminderScheduled   → 安排30分钟后推送提醒
       ↓
[D1] ReminderPushedEvent             → 推送注册引导
       ↓
[D2] UserRegisteredEvent             → 注册成功
       ↓
[D1] WelcomeJourneyStarted           → 启动7天新手引导旅程
```

#### 4.1.3 价值链分析

从用户视角出发，平台的价值链可以表述为：

```
触达价值         认知价值         决策价值         执行价值         陪伴价值
(D1+D6)    →    (D6)      →    (D3)      →    (D4+D5)   →    (D3+D1)

"让我知道      "让我理解       "帮我选择       "帮我买入       "帮我管好
 养老金"        养老金"         适合的方案"     和管理"         我的养老金"
```

每个环节的价值实现依赖于上游环节的输出，形成链式依赖。如果某一环断裂（例如用户没有完成风险测评），后续所有价值均无法交付。因此，**用户旅程的流畅度**比单一功能的质量更重要。

### 4.2 关键协作模式

| 协作模式 | 涉及域 | 交互方式 | 说明 |
|----------|--------|----------|------|
| 画像驱动营销 | D2 → S1 → D1 | 数据流 + 事件 | 用户属性/行为 → 标签计算 → 精准触达 |
| 测评驱动配置 | D2 → D3 → D4 | 事件 + API调用 | 风险等级 → 配置算法 → 产品匹配 |
| 配置驱动交易 | D3 → D5 → D4 | 事件 + API调用 | 配置方案 → 批量申购 → 产品确认 |
| 持仓驱动再平衡 | D5 → S1 → D3 → D1 | 定时任务 + 事件 | 持仓数据 → 偏离计算 → 再平衡建议 → 推送用户 |
| 内容驱动获客 | D6 → D1 | 数据流 | 内容阅读行为 → 兴趣标签 → 个性化推荐 |

## 六、商业计划书（BP）

### 6.1 市场机会

**市场规模**：中国个人养老金制度覆盖约4亿城镇职工，按年缴费上限12,000元计算，潜在可缴费资金规模达4.8万亿元/年。按当前约30%的实际缴存率，尚有约3.4万亿元的"沉睡"市场等待激活。

**竞争格局**：当前市场由三类玩家主导——银行系（招行、工行等，依托网点和客户基础）、互联网系（蚂蚁财富、腾安，依托流量和数据）、传统基金公司直销。但在"养老金垂直投顾"这一细分赛道上，尚无具有明显优势的头部玩家。现有智能投顾产品（如帮你投、摩羯智投）定位于全品类理财，养老金只是一个子场景，缺乏深度。

**时间窗口**：2024年个人养老金制度全国推广后，用户教育成本大幅下降。当前正处于"开户量高但激活率低"的关键节点，谁能率先解决"缴存冷"的问题，谁就能占据这一赛道的用户心智。

### 6.2 商业模式

平台采用"基础免费 + 增值服务收费 + 机构分成"的三层收入模型：

**收入层一：基金销售尾随佣金** — 作为持牌销售机构的合作渠道，从用户持有基金产品的管理费中获得30%-50%的尾随佣金分成。按AUM的0.15%-0.30%/年计算。这是初期最主要的收入来源。

**收入层二：智能投顾服务费** — 对使用智能投顾服务的用户按AUM的0.20%-0.50%/年收取。初期（前12个月）对AUM < 10万元的用户免收，以降低使用门槛。

**收入层三：机构服务费** — 向接入平台的金融机构提供用户画像洞察、产品表现分析等增值数据服务，按年收取服务费。

**收入层四：会员增值服务** — 黄金及以上会员（299-999元/年）享受专属投顾报告、人工顾问咨询、高端活动等权益。
